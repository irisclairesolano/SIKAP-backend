================================================================
CONTEXT & CONSTRAINTS
================================================================

- Database is already created in Supabase (all 13 tables, ENUMs, RLS, seeds). Do NOT run migrations or db:seed for these tables — Supabase schema is the source of truth.
- Laravel migrations are only needed for: personal_access_tokens (Sanctum), jobs (queue), cache, sessions tables.
- All DB access uses service_role key through Laravel backend. RLS blocks all direct PostgREST access.
- Email is sent via Resend API using Laravel's native Resend mailer (MAIL_MAILER=resend). NOT SMTP.
- SMS is sent via Semaphore Philippines API (https://api.semaphore.co/api/v4/messages).
- Semaphore API key is NOT yet available — leave SEMAPHORE_API_KEY=<your-semaphore-api-key> as placeholder in .env.example. The SemaphoreService must be implemented but the key will be filled in later.
- File storage is Supabase Storage (S3-compatible), accessed via Guzzle HTTP — no SDK.
- Auth uses Laravel Sanctum bearer tokens (stateless, for mobile API).
- No frontend — pure REST API only.

STEP 1 — PROJECT SETUP
================================================================

Run these commands to scaffold the Laravel 12 project:

    composer create-project laravel/laravel sikap-backend
    cd sikap-backend
    composer require laravel/sanctum
    composer require guzzlehttp/guzzle
    composer require resend/resend-laravel
    php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
    php artisan key:generate

Then run ONLY these migrations (Supabase already has the app tables):

    php artisan queue:table
    php artisan cache:table
    php artisan session:table
    php artisan migrate

FILE: app/Services/ApplicationService.php
----------------------------------------------------------------
Methods:

    apply(User $worker, JobPost $job, ?string $coverNote): Application
    - Guard: job->status === 'open' (else 422)
    - Guard: applications count < 50 (else 422 'Application limit reached.')
    - Guard: no duplicate application (else 422 'Already applied to this job.')
    - Create Application with status=pending, applied_at=now()

    sendJobRequest(Application $app): void
    - Guard: app->status === 'pending' (else 422)
    - Update: status=pending_negotiation, references_revealed=true
    - SMS to worker: Stage 2 template

    confirmHire(Application $app, float $price): void
    - Guard: app->status === 'pending_negotiation' (else 422)
    - DB::transaction with lockForUpdate:
        - Check job->accepted_count < job->slots (else 422 'All slots are currently locked.')
        - Update app: status=employer_confirmed, final_agreed_price, slot_locked_at=now(), employer_confirmed_at=now()
        - job->increment('accepted_count')
    - Dispatch SendHireReminderJob::dispatch($app)->delay(now()->addHours(24))

    acceptOffer(Application $app): void
    - Guard: app->status === 'employer_confirmed' (else 422)
    - Update app: status=accepted, contact_revealed=true, responded_at=now()
    - If job->accepted_count >= job->slots: job->update(['status' => 'closed_in_progress'])

    rejectOffer(Application $app): void
    - Guard: app->status === 'employer_confirmed' (else 422)
    - DB::transaction:
        - Update app: status=rejected, responded_at=now()
        - job->decrement('accepted_count')
    - SMS to employer: Worker Rejected template

    withdraw(Application $app): void
    - Guard: app->status === 'pending' (else 422 'Can only withdraw a pending application.')
    - Update app: status=withdrawn

    cancelHire(Application $app): void
    - Guard: app->status in [employer_confirmed, pending_negotiation] (else 422)
    - DB::transaction:
        - If status was employer_confirmed: job->decrement('accepted_count')
        - Update app: status=rejected, responded_at=now()
    - SMS to worker: Employer Cancelled template

    markJobComplete(JobPost $job): void
    - Guard: job->status === 'closed_in_progress' (else 422)
    - Update job: status=completed, rating_window_expires_at=now()->addDays(7)
    - Update all accepted applications for this job: status=completed

    flagOffline(Application $app): void
    - Guard: app->status === 'accepted' (else 422)
    - Update app: status=completed
    - Check if all accepted apps for this job are completed -> markJobComplete

FILE: app/Services/AnalyticsService.php
----------------------------------------------------------------
Methods:
    all(string $from, string $to): array
    - Returns array with keys: skill_demand, application_volume, registration_trends, skill_distribution, geographic_activity

    skillDemand(string $from, string $to): array
    SQL:
        SELECT jp.category, COUNT(jp.id) AS total_postings
        FROM job_posts jp
        WHERE jp.deleted_at IS NULL AND jp.created_at BETWEEN :from AND :to
        GROUP BY jp.category ORDER BY total_postings DESC

    applicationVolume(string $from, string $to): array
    SQL:
        SELECT jp.id AS job_post_id, jp.reference_number, jp.title, jp.category,
               COUNT(a.id) AS total_applications,
               SUM(CASE WHEN a.status='pending' THEN 1 ELSE 0 END) AS pending,
               SUM(CASE WHEN a.status='pending_negotiation' THEN 1 ELSE 0 END) AS in_negotiation,
               SUM(CASE WHEN a.status='employer_confirmed' THEN 1 ELSE 0 END) AS confirmed,
               SUM(CASE WHEN a.status='accepted' THEN 1 ELSE 0 END) AS accepted,
               SUM(CASE WHEN a.status='completed' THEN 1 ELSE 0 END) AS completed,
               SUM(CASE WHEN a.status='rejected' THEN 1 ELSE 0 END) AS rejected,
               SUM(CASE WHEN a.status='withdrawn' THEN 1 ELSE 0 END) AS withdrawn
        FROM job_posts jp
        LEFT JOIN applications a ON a.job_post_id = jp.id
        WHERE jp.deleted_at IS NULL AND jp.created_at BETWEEN :from AND :to
        GROUP BY jp.id, jp.reference_number, jp.title, jp.category
        ORDER BY total_applications DESC LIMIT 50

    registrationTrends(string $from, string $to): array
    SQL:
        SELECT TO_CHAR(created_at, 'YYYY-MM') AS month, role, municipality, COUNT(id) AS registrations
        FROM users
        WHERE created_at BETWEEN :from AND :to
        GROUP BY month, role, municipality
        ORDER BY month ASC, registrations DESC

    skillDistribution(): array
    SQL:
        SELECT s.category, s.name AS skill_name, COUNT(ws.worker_profile_id) AS worker_count
        FROM worker_skill ws
        JOIN skills s ON s.id = ws.skill_id
        JOIN worker_profiles wp ON wp.id = ws.worker_profile_id
        JOIN users u ON u.id = wp.user_id
        WHERE u.is_suspended = FALSE AND u.verification_status = 'approved'
        GROUP BY s.category, s.name
        ORDER BY s.category ASC, worker_count DESC

    geographicActivity(string $from, string $to): array
    SQL:
        SELECT jp.municipality, jp.barangay,
               COUNT(DISTINCT jp.id) AS job_postings,
               COUNT(a.id) AS total_applications
        FROM job_posts jp
        LEFT JOIN applications a ON a.job_post_id = jp.id
        WHERE jp.deleted_at IS NULL AND jp.created_at BETWEEN :from AND :to
        GROUP BY jp.municipality, jp.barangay
        ORDER BY jp.municipality ASC, job_postings DESC

FILE: app/Http/Controllers/Auth/AuthController.php
----------------------------------------------------------------

POST /api/v1/auth/register
Validation:
    name: required|string|max:255
    email: required|email|unique:users,email
    password: required|min:8|confirmed
    role: required|in:worker,employer
    phone: required|string|max:20
    barangay: required|string
    municipality: required|string
    referrer_contact: nullable|string|max:20
Logic:
    1. Create user (bcrypt password)
    2. If referrer_contact: create Referral (encrypt referrer_contact with encrypt())
    3. Generate 6-digit OTP (str_pad(random_int(0,999999), 6, '0', STR_PAD_LEFT))
    4. Store EmailOtp: otp_hash=bcrypt(otp), expires_at=now()+10min
    5. Mail::to($user->email)->send(new OtpMail($otp))
    6. app(SemaphoreService::class)->send($user->phone, OTP template)
    7. Return 201: {message: 'Account created. OTP sent.'}

POST /api/v1/auth/verify-otp
Validation: email: required|email, otp: required|digits:6
Logic:
    1. Find user by email
    2. Find latest EmailOtp where user_id=user->id AND expires_at > now()
    3. If not found OR !Hash::check($otp, $record->otp_hash): abort 422 'OTP expired or invalid. Request a new one.'
    4. user->update([email_verified_at => now()])
    5. Delete OTP record
    6. Return 200: {message: 'Email verified. Please upload your government ID.'}

POST /api/v1/auth/resend-otp
Validation: email: required|email
Logic:
    1. Find user by email
    2. Find latest OTP — if exists and created_at > now()-1min: abort 429 'Please wait before requesting a new OTP.'
    3. Delete old OTPs for user
    4. Generate new OTP, store, send email + SMS
    5. Return 200: {message: 'OTP resent.'}

POST /api/v1/auth/login
Validation: email: required|email, password: required
Logic:
    1. Find user by email
    2. If !user OR !Hash::check(password, user->password): abort 401 'Invalid credentials.'
    3. If !user->email_verified_at: abort 403 'Email not verified.'
    4. If user->is_suspended: abort 403 'Account suspended.'
    5. Create Sanctum token: $token = $user->createToken('auth_token')->plainTextToken
    6. Return 200: {token, token_type: 'Bearer', user: {id, name, email, role, verification_status, verification_badge}}

POST /api/v1/auth/upload-id [auth:sanctum]
Validation: id_file: required|file|mimes:jpeg,png|max:5120
Logic:
    1. $path = 'ids/' . $user->id . '_' . time() . '.' . $file->extension()
    2. $url = app(SupabaseStorageService::class)->upload($file, $path)
    3. user->update([document_url => $url, verification_status => 'pending'])
    4. Return 200: {message: 'ID uploaded. Account is pending admin approval.'}

POST /api/v1/auth/logout [auth:sanctum]
Logic:
    1. $request->user()->currentAccessToken()->delete()
    2. Return 200: {message: 'Logged out.'}

STEP 14 — BUSINESS RULES SUMMARY (enforce in code)
================================================================

- Max 50 applications per job post
- Max 5 active jobs (open + closed_in_progress) per employer at once
- One application per worker per job (enforced by DB UNIQUE constraint)
- One review per party per job (enforced by DB UNIQUE constraint)
- Reviews only allowed within 7-day window after job completion (rating_window_expires_at)
- Slot locking: use DB::transaction + lockForUpdate when incrementing accepted_count
- OTP: store bcrypt hash only, delete immediately after verification, rate-limit resend to 1/min
- Worker email: NEVER disclosed at any stage — not even to admin via API
- Worker phone: only when contact_revealed = true (Stage 4)
- Character references: only when references_revealed = true (Stage 2)
- Semaphore phone format: replace leading 0 with 63
